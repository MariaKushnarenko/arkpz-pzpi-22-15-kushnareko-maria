1. Business Requirements
1.1 Background
      Сучасні притулки для тварин стикаються з численними викликами у забезпеченні належного догляду за тваринами. По-перше, через обмежені ресурси та високу завантаженість персоналу виникає необхідність у більш ефективних методах моніторингу стану тварин. Багато притулків не мають можливості вести постійний моніторинг здоров'я та активності тварин, що призводить до труднощів у виявленні проблем на ранній стадії.
      Крім того, зростає усвідомлення важливості використання технологій для підвищення якості життя тварин. Впровадження інноваційних технологій, таких як IoT, може значно поліпшити спостереження за тваринами, автоматизувати процеси догляду і забезпечити персоналу можливість швидше реагувати на зміни у стані здоров’я тварин.
      Ця ситуація створює необхідність у розробці програмної системи для моніторингу стану тварин у притулках, яка б забезпечила інтеграцію технологій, покращила б управлінські процеси та сприяла благополуччю тварин. Система стане відповіддю на виклики сучасності, дозволяючи притулкам оптимізувати свої ресурси та підвищити рівень догляду за тваринами.
      
1.2 Business Opportunity
      На ринку догляду за тваринами існує значна можливість для впровадження інноваційних рішень, які б підвищили ефективність управління притулками. Сьогодні більшість існуючих систем моніторингу тварин обмежені простими функціями спостереження, такими як відеокамери або ручні записи. Ці рішення часто є неінтегрованими та не забезпечують реального часу спостереження за станом здоров'я і активності тварин. Програмна система, що поєднує IoT-технології та аналітику, здатна заповнити цю прогалину, надаючи притулкам потужний інструмент для моніторингу та управління здоров’ям тварин, автоматизації процесів годування і покращення якості життя тварин.
      Запропоноване рішення відповідає сучасним тенденціям у сфері автоматизації та використання великих даних у догляді за тваринами. З розвитком технологій IoT та аналітиці даних, притулки отримують можливість застосовувати нові інструменти для оптимізації своєї роботи. Це не тільки дозволяє зменшити навантаження на персонал, але й підвищити якість догляду, що важливо для залучення донорів і підтримки репутації. В умовах постійно зростаючого попиту на ефективні рішення в догляді за тваринами, система моніторингу має потенціал стати лідером на ринку, забезпечуючи користувачів інноваційними, доступними та надійними інструментами для управління станом тварин.
      
      
      
      
      
2. Vision of the Solution
2.1 Vision Statement
      Програмна система для моніторингу стану тварин у притулках забезпечить постійний, зручний та доступний контроль за здоров'ям і добробутом тварин, сприяючи підвищенню стандартів догляду та управління притулками. Ця система дозволить автоматизувати рутинні процеси, забезпечить моніторинг життєво важливих показників у реальному часі, а також створить умови для оперативного реагування на будь-які зміни стану тварин. Завдяки інтуїтивному інтерфейсу та можливостям IoT, система зробить догляд за тваринами більш ефективним і прозорим, дозволяючи співробітникам зосередитися на пріоритетних завданнях. Ми прагнемо створити продукт, який стане незамінним інструментом для притулків, покращуючи якість життя тварин і встановлюючи нові стандарти для догляду за ними.
      
2.2 Major Features
      MF-1: Моніторинг життєво важливих показників тварин (температура, серцевий ритм тощо) через IoT-пристрої.
      MF-2: Перегляд загального стану тварин у притулку з можливістю швидкого сортування за станом.
      MF-3: Реєстрація користувачів з розподілом ролей (адміністратори, доглядачі).
      MF-4: Налаштування сповіщень для персоналу про критичні зміни стану тварин.
      MF-5: Перегляд історії медичних показників кожної тварини для аналізу динаміки.
      MF-6: Підтримка мультиінтерфейсу для користувачів (українська та англійська мови).
      MF-7: Інтерактивна панель адміністрування для керування користувачами та пристроями.
      MF-8: Захист персональних даних відповідно до сучасних стандартів безпеки.
      MF-9: Функція експорту та імпорту даних тварин для забезпечення створення звітів.


3. Scope and Limitations
3.1 Scope of Initial Release
      Back-end: Обробка та зберігання даних
      Back-end виконує ключову функцію у зборі, обробці та зберіганні даних від IoT-пристроїв та забезпечує бізнес-логіку для системи:
1. Обробка та зберігання даних IoT: Постійний збір даних від сенсорів, включаючи показники здоров'я (температура, пульс, активність) та стан тварин у режимі реального часу.
2. Профілі тварин: Зберігання інформації про кожну тварину, включаючи її біографічні дані, історію здоров'я, та поточні показники.
3. Сповіщення про критичні зміни: Автоматичне генерування та надсилання сповіщень про критичні зміни в стані тварини, що потребує швидкого втручання персоналу.
4. Відеоаналітика та управління годівницею через API: Підтримка інтеграції з відеоаналітикою для моніторингу активності тварин та API для автоматичного управління годівницями. Back-end забезпечує доступ до команд управління годівницею та отримання відеопотоку для аналітичних цілей.
5. Адміністрування користувачів та доступу: Управління обліковими записами користувачів, їх ролями та правами доступу для забезпечення конфіденційності даних та зручного адміністрування системи.
      Front-end: Веб-інтерфейс для адміністраторів
      Веб-інтерфейс надає адміністраторам доступ до повного функціоналу для моніторингу та керування системою:
1. Профілі тварин: Відображення детальної інформації про кожну тварину, включаючи її поточні показники здоров'я, активність та історію захворювань.
2. Аналітика та звітність: Генерація аналітичних даних для візуалізації стану здоров'я тварин з використанням графіків та звітів.
3. Налаштування годування: Надання можливості адміністраторам вручну налаштовувати параметри годування для тварин та запускати процеси годування за потреби.
4. Перегляд сповіщень та критичних змін: Відображення всіх активних сповіщень, що вимагають уваги, а також перегляд історії критичних подій, щоб персонал міг оперативно реагувати на будь-які надзвичайні ситуації.
5. Мовна підтримка (локалізація): Інтерфейс веб-додатку буде доступний українською та англійською мовами для міжнародного використання.
      IoT-пристрої: Збір та передача даних
      IoT-пристрої забезпечують прямий зв’язок між тваринами та системою:
1. Сенсори здоров'я та активності: Збір даних про здоров'я тварин, включаючи температуру, пульс, активність та інші показники, які автоматично передаються на back-end для обробки та моніторингу.
2. Годівниці з автоматичним управлінням: Годівниці, що керуються через API, дозволяють налаштовувати графік годування та забезпечувати необхідне живлення для тварин, згідно з даними системи. Вони отримують команди від back-end та забезпечують виконання інструкцій в режимі реального часу.
      Mobile: Мобільний додаток для персоналу
      Мобільний додаток створений для полегшення доступу персоналу до даних та управління тваринами з будь-якої точки:
1. Перегляд даних про здоров'я та активність: Мобільний інтерфейс дозволяє персоналу переглядати актуальні показники здоров'я тварин у режимі реального часу та оцінювати їх стан.
2. Отримання сповіщень про критичні стани: Автоматичні сповіщення про критичні показники здоров'я надсилаються на мобільні пристрої персоналу, що дозволяє негайно вживати необхідних заходів.
3. Управління годуванням у реальному часі: З мобільного додатку персонал може віддалено налаштовувати параметри годування або запускати процес годування для певних тварин, не підходячи до годівниць фізично.
      Захист персональних даних та інформаційна безпека
      Для забезпечення безпеки конфіденційних даних системи впроваджуються наступні заходи:
1. Шифрування даних: Всі дані, включаючи інформацію про користувачів та показники здоров'я тварин, зберігаються в зашифрованому вигляді для забезпечення захисту інформації.
2. Аутентифікація та розмежування доступу: Всі користувачі проходять аутентифікацію з використанням безпечного процесу входу, який також забезпечує контроль доступу на основі ролей.
      Бізнес-логіка: Реалізація основних процесів
      Основні функції бізнес-логіки для забезпечення коректної роботи всіх модулів системи включають:
1. Автоматичне управління станом здоров'я: Бізнес-логіка back-end забезпечує аналіз даних, що надходять від IoT-сенсорів, для своєчасного виявлення відхилень в стані здоров'я тварин.
2. Обробка команд годування: Система обробляє автоматичні команди для годівниць, враховуючи попередньо налаштовані графіки або вказівки персоналу.
3. Аналіз та обробка відеоаналітики: Використання алгоритмів для відеоаналітики дозволяє отримувати інформацію про поведінкові зміни, які можуть сигналізувати про проблеми зі здоров'ям або підвищену активність тварин.

3.2 Scope of Subsequent Releases
      1. Можливість інтеграції з іншими IoT-пристроями: розширення функціоналу для підключення різних типів датчиків і пристроїв, що дозволить отримувати більш детальну інформацію про стан тварин.
      2. Покращений алгоритм аналізу даних: розробка алгоритмів для більш глибокого аналізу даних і прогнозування стану здоров’я тварин на основі історичних даних.
      3. Можливість донорства та фінансування: інтеграція функцій для збору коштів на підтримку притулків через платформу, що дозволить залучати більше ресурсів для допомоги тваринам.
      4. Залучення волонтерів: функціонал для залучення волонтерів, що дозволить координувати їхню діяльність та отримувати відгуки про тварин, з якими вони працюють.
      5. Інтеграція з іншими системами: можливість інтеграції з існуючими системами управління притулками, ветеринарними базами даних або CRM-системами для покращення обміну інформацією.
      6. Мультимовна підтримка: на початковому етапі не планується реалізація підтримки декількох мов інтерфейсу, що обмежить доступність системи для користувачів, які не володіють мовою, на якій вона буде запущена.

4. Business Context
4.3 Operating Environment
1. Back-end (серверна частина)
1. Мова програмування: C# (C-Sharp) 
C# є основною мовою програмування для реалізації серверної частини додатку. 

2. Фреймворк: .NET Core
2. Front-end (клієнтська частина)
1. Мова програмування: JavaScript
JavaScript є основною мовою для реалізації фронтенду, завдяки її гнучкості та широкій підтримці сучасних фреймворків.

2. Фреймворк: React

3. CSS фреймворк: Bootstrap
Для швидкої розробки адаптивного дизайну буде використаний фреймворк Bootstrap, який дозволяє створювати стильні та адаптивні веб-сторінки з мінімальними зусиллями.
3. Мобільний додаток (Android)
1. Мова програмування: Kotlin
Kotlin є основною мовою програмування для розробки Android-додатків. 

2. Android SDK: Android Studio

3. UI фреймворк: Jetpack Compose
4. IoT-пристрої
1. Мова програмування: C++
Для програмування IoT-пристроїв буде використовуватися мова C++, яка оптимальна для розробки на низькому рівні та дозволяє працювати з різними мікроконтролерами та сенсорами. 

2. Платформа: Arduino / Raspberry Pi (в залежності від типу пристрою)
Для збору даних із сенсорів та керування годівницями будуть використовуватись платформи Arduino або Raspberry Pi. 
5. База даних
1. Система керування базою даних (СУБД): Microsoft SQL Server
Для зберігання та обробки даних про тварин, їх здоров'я та активність буде використовуватись Microsoft SQL Server. Ця реляційна СУБД є надійною, масштабованою та ефективною для зберігання великих обсягів структурованих даних.

6. Інші інструменти
1. Система контролю версій: Git
Git буде використовуватися для контролю версій та спільної роботи над кодом. Репозиторії будуть зберігатися на платформі GitHub або GitLab.
